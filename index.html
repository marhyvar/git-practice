Configuring Git

After installing Git, you should configure at least these:

$ git config --global user.name "Your Name"

$ git config --global user.email youremailaddress@example.com

These are included in every Git commit you make.

If you do not want to use Git's default editor, you should configure your own preferred editor:

$ git config --global core.editor editor-of-choice

Note, on a Windows system, you need to specify the full path to the executable file of the editor!

Here's how you can check your settings:

$ git config --list

Setting up a Git repository

Make a directory for your project. Go to your project directory and type:

$ git init

This adds a .git subdirectory to your project. You can start building your project or, you can clone an existing Git repository:

$ git clone https://github.com/example/project

Make the cloned repository your remote repository by typing:

$ git remote add origin https://github.com/example/project

Working with Git

After making changes you can type:

$ git status

This will show you your untracked and modified files. If you want to include these to the next commit:

$ git add .

After this you can make a commit of the changes you made to the project:

$ git commit -m "write a good commit message here"

You can push these changes to the remote repository:

$ git push -u origin master

The "-u" sets the remote repository. When you want to push your changes again, you can just type:

$ git push

Removing a file:

$ rm name-of-the-file

$ git rm name-of-the-file

$ git commit -m "remove name-of-the-file"

Discarding changes in the working directory (before git add):

$ git checkout -- name-of-the-file

Undoing git add (staging)

$ git reset HEAD file-to-be-unstaged

Branching

Create a new branch and immediately switch to this new branch:

$ git checkout -b name-of-the-branch

You can also create a new branch by typing:

$ git branch name-of-the-branch

$ git branch -m old-name new-name
